    Client Site API

    reference
    Use the Client Site API to create and manage sites for your clients. Also use this API to retrieve a user_id and site_id needed to access the rest of the Cloud API.

    Fields

    The following table shows all fields that exist for this API, those that are returned when you retrieve a list, those that are required for POST. All fields are returned when you retrieve a single item.

    Name Description Type List Required
    reseller_client_id ID of a cleint, unique to the reseller.​ string
    site_id ID of a Weebly site, unique to the currently authenticated user​ string
    user_id ID of a Weebly user.​ string
    site_domain Fully qualified domain name for the site, for example www.samplesite.com​ string X
    site_title Title given to a site. string
    brand_name Your brand name, as set in the Cloud Admin. If not set on the site level, the brand_name set in the Cloud Admin is used. Passing a NULL value will ensure the site uses the value defined in the Cloud Admin. string
    brand_url Full URL to your website, as set in the Cloud Admin. The link in the footer of a published site will go to this site. If not set on the site level, the brand_url set in the Cloud Admin is used. Passing a NULL value will ensure the site uses the value defined in the Cloud Admin. string
    plan_id Unique ID for the plan for this site. string
    term Amount of time, in months, that the plan is good for. string
    time_zone Timezone used on the site. Valid values can be found here. string
    time_format Format used for time on the site. Values are 12 and 24. string
    date_format Format used for dates on the site. Values are n/j/Y for month/day/year and j/n/Y for day/month/year. string
    created_date Date the client site was created. Unix GMT Timestamp <
    updated_date Date the client site was last updated. Unix GMT Timestamp
    Note: Any fields returned that are not documented are currently unsupported and can be safely ignored.
